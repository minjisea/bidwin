## 1. 프로젝트 개요

### 팀명 및 프로젝트명

| 항목 | 내용 |
|------|------|
| **프로젝트명** | BID-AI — 공공입찰 낙찰률 예측 및 AI 제안서 생성 시스템 |
| **팀명** | BID-AI 팀 |
| **프로젝트 기간** | 2026년 3월 초 ~ 2026년 4월 (4주) |
| **개발 형태** | 자율 프로젝트 (사내 자율 공모) |

### 추진 배경

공공입찰 실무 현장에서 반복적으로 발생하는 3가지 Pain Point를 데이터와 AI로 해결하고자 프로젝트를 시작했다.

| Pain Point | 현황 | 문제점 |
|-----------|------|--------|
| 공고 선별의 어려움 | 수천 건의 입찰공고 중 낙찰 가능성 높은 건 선별 | 경험·감에 의존, 정량 기준 없음 |
| 낙찰률 예측 불가 | 배정예산 대비 적정 투찰금액 산정 | 업종·기관별 낙찰 패턴 분석 불가 |
| 제안서 작성 비효율 | 제안서 1건당 평균 3~5일 소요 | 반복 구조의 수작업, 공고 특성 반영 어려움 |

### 추진 목적

조달청 공공데이터포털의 입찰공고 결과내역(공개 데이터)을 활용하여 **낙찰률 예측 + AI 제안서 자동 생성 파이프라인**을 구축하고, 데이터 기반 입찰 의사결정 플랫폼을 개발한다.

## 2. 개발 목표 및 구현 범위

### 3가지 핵심 목표

#### 목표 1: 낙찰률 예측 모델 (LightGBM + TF-IDF)
- 배정예산 대비 낙찰금액 비율(투찰율)을 ML로 예측
- 최적 투찰가 산정 근거 제공
- **달성 결과**: MAE 2.24, R² 0.55 (Baseline 대비 MAE 31% 개선)

#### 목표 2: AI 제안서 자동 생성 (Claude API)
- 공고 정보 입력만으로 맞춤형 제안서 초안 자동 생성
- Few-shot + Chain-of-Thought 프롬프트 엔지니어링 적용
- **달성 결과**: 4개 섹션(사업이해/추진전략/기술제안/수행계획) 구조화 자동 생성

#### 목표 3: 입찰 적합도 스코어링 (Scoring Pipeline)
- 공고별 수주 적합도를 정량 점수(0~100)로 산출
- 업종 필터 → Fit Score 순 정렬 → 상위 N건 자동 추천
- **달성 결과**: Spearman 상관계수 0.938로 실제 낙찰 순위와 고일치 달성

### 구현 범위

| 기능 | 구현 여부 | 비고 |
|------|---------|------|
| 조달청 CSV 데이터 수집·전처리 | ✅ 완료 | 44,896건 |
| Feature Engineering (14종 파생 피처) | ✅ 완료 | TF-IDF 300차원 포함 |

| 다중 모델 비교·통계 검증 | ✅ 완료 | 3×5-Fold CV, Paired t-test |
| AI 제안서 프롬프트 자동 생성 | ✅ 완료 | Claude API 연동 구조 |

## 4. 개발 에피소드

### 개발 과정 (AI 활용)

**Claude Code 활용 내역**
- 코드 스캐폴딩: 전처리·피처 엔지니어링·모델링 파이프라인 기본 구조 생성
- 디버깅 보조: pandas DataFrame 연산 오류, TF-IDF 희소행렬 처리 이슈 해결
- 리뷰: Streamlit app.py 성능 최적화 (`@st.cache_data` 적용)
- 실험 자동화: 3×5-Fold CV 반복 코드, 결과 CSV 저장 로직 생성

**Claude API 활용 내역**
- 프롬프트 엔지니어링: 공공입찰 제안서 작성에 특화된 시스템 프롬프트 설계
- Few-shot 예시: 우수 제안서 구조를 예시로 제공하여 출력 품질 향상
- Chain-of-Thought: 분석→전략→작성 단계적 생성으로 일관성 확보

### 예상 외 에피소드

| 에피소드 | 내용 |
|---------|------|
| **TF-IDF의 낮은 기여도** | 텍스트 300차원 피처보다 파생 피처 4개가 성능에 더 중요함을 Ablation Study로 확인. 기대와 다른 결과였으나 도메인 특성상 수치 패턴이 텍스트보다 낙찰률과 더 직결됨을 이해 |
| **스코어링 가중치 설계** | 초기 직관적 가중치(낙찰률 0.45/예산 0.30/경쟁 0.25)보다 LightGBM Feature Importance 기반 가중치(0.487/0.268/0.246)가 Spearman 0.938로 더 우수함을 실험으로 증명 |
| **CPU 환경의 제약** | GPU 없이 LightGBM으로 충분한 성능 달성. 오히려 XGBoost 대비 4~5배 빠른 학습 속도가 반복 실험에 유리 |
| **R² = 0.55 해석** | 낮아 보이지만 공공 입찰 데이터 특성상(비공개 정보, 담합, 전략적 투찰 등) 공개 데이터만으로 달성한 수치로는 의미 있는 수준임을 문헌 비교로 검증 |

### 팀워크

- 1인 개발 프로젝트로 전체 파이프라인(데이터 수집~UI)을 단독 설계·구현
- Claude Code를 개발 파트너로 활용하여 4주 내 End-to-End 시스템 완성

### 주요 기능

#### Dashboard 페이지
- 공고 현황 KPI 카드 (총 공고수, 평균 투찰율, 평균 예산, 평균 적합도)
- 투찰율 분포 히스토그램
- 공공조달분류 TOP 10 차트
- 적합도 TOP 20 공고 테이블

#### Prediction 페이지
- 공고명·배정예산·분류·참여자수·업종조건 입력 폼
- 예상 투찰율 + 적합도 점수 + 등급(A/B/C/D) 산출
- 세부 점수 분해 테이블 (낙찰률/예산/경쟁 강도)
- 유사 공고 비교 테이블

#### Proposal 페이지
- 공고 정보 입력 → Claude API용 상세 프롬프트 자동 생성
- 5개 섹션(사업이해/추진전략/기술제안/수행계획/차별화) 구조화
- claude.ai 붙여넣기로 즉시 제안서 초안 생성 가능
- 유사 공고 참고 데이터 제공

#### Analysis 페이지
- 모델 실험 결과 비교 테이블 + MAE 비교 차트
- Feature Importance TOP 20 시각화
- 참여자수·예산 구간별 평균 투찰율 분석
- EDA 시각화 이미지 자동 로드
